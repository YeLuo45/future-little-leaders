# 首页多宝宝任务隔离功能

## 功能介绍

首页（`index.vue`）现已支持多宝宝任务隔离功能，用户可以在页面顶部切换不同宝宝，系统会自动展示对应宝宝的任务列表和完成情况。该功能与"我的"页面的宝宝选择器联动，保证整个应用的一致性。

## 核心功能

1. **宝宝选择器**
   - 在首页顶部添加宝宝选择器
   - 可快速切换当前操作的宝宝
   - 样式与整体UI保持一致，融入顶部区域

2. **任务数据隔离**
   - 按宝宝ID（`babyId`）筛选任务
   - 为所有任务添加宝宝ID属性，新增任务自动关联当前宝宝
   - 历史任务无宝宝ID时可查看（向后兼容）

3. **全局同步**
   - 通过 `babyChanged` 事件监听全局宝宝切换
   - 自动刷新当前宝宝任务列表
   - 确保所有页面数据一致性

## 数据结构

```js
// 任务对象新增babyId属性
{
  // ... 原有字段
  babyId: "baby_1647862420123" // 关联的宝宝ID
}
```

## 兼容性与旧数据迁移

- 无 `babyId` 的旧任务仍可显示（`!task.babyId || task.babyId === currentBabyId.value`）
- 打卡完成时自动关联到当前宝宝
- 新增任务时自动关联到当前宝宝

## 使用方式

1. 首页顶部下拉选择宝宝
2. 添加任务自动关联到当前宝宝
3. 在"我的"页面切换宝宝后，首页自动同步

## 技术方案说明

首页任务区多列布局切换实现思路：

1. 参考商城页(shop.vue)的布局切换方案，增加layoutType变量和setLayoutType方法，支持1/2/3列切换。
2. 在任务区上方增加布局切换按钮，按钮样式与商城页一致。
3. 任务卡片外层增加.task-list容器，根据layoutType动态切换不同的列数样式。
4. 迁移并适配shop.vue的多列布局样式到index.vue，适配.task-item和.task-card。
5. 保证不同列数下任务卡片宽度自适应，移动端体验良好。
6. 相关代码、样式、注释同步完善。
