# 积分记录页面实现说明

## 功能概述
积分记录页面用于显示当前选中宝宝的所有积分收支记录，包括获得积分和消费积分的详细信息。该页面通过Pinia状态管理器(pointsStore)管理积分数据，支持宝宝切换，并根据不同宝宝显示对应的积分记录。

## 主要功能
1. 显示当前宝宝的积分收支记录
2. 支持切换宝宝查看不同宝宝的积分记录
3. 区分积分收入和支出，使用不同颜色标记
4. 按时间倒序排列，最新记录显示在前
5. 兼容深色模式
6. 显示积分变更的详细信息，包括任务名称或商品名称

## 技术实现
- 使用Pinia状态管理：通过`usePointsStore`和`useBabyStore`管理积分和宝宝数据
- 组合式API：使用Vue 3的`<script setup>`组合式API实现
- 响应式数据：利用`ref`, `computed`等实现数据响应式
- 事件总线：通过uni.$on和uni.$emit实现跨组件通信
- 统一积分管理：所有积分变更操作都通过pointsStore管理，确保记录完整

## 数据来源
积分记录主要来自两个源头：
1. 完成任务获得积分：记录任务名称和获得的积分
2. 商城兑换商品消费积分：记录商品名称和消费的积分

## 数据结构
积分记录格式：
```js
{
  id: String,              // 记录ID
  babyId: String,          // 宝宝ID
  points: Number,          // 积分数量
  description: String,     // 记录描述（包含任务名称或商品名称）
  type: String,            // 类型：'income'(收入)或'expense'(支出)
  createdAt: String        // 创建时间ISO格式
}
```

## 事件处理
- `refreshBabyList`: 刷新宝宝列表
- `babyChanged`: 宝宝切换事件
- `pointsUpdated`: 积分更新事件
