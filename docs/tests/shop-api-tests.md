# 积分商城API接口测试用例

## 1. 获取商品列表接口测试

### 测试用例 1.1 - 获取所有商品列表

#### 前置条件
- 用户已登录，拥有有效的token

#### 测试步骤
1. 使用GET方法请求 `/api/shop/products` 接口
2. 请求头中携带有效的Authorization信息

#### 预期结果
- 返回状态码200
- 响应数据包含商品列表
- 返回的JSON格式符合API文档规范
- 分页信息正确

### 测试用例 1.2 - 按分类筛选商品

#### 前置条件
- 用户已登录，拥有有效的token
- 系统中已存在"学习用品"分类的商品

#### 测试步骤
1. 使用GET方法请求 `/api/shop/products?category=学习用品` 接口
2. 请求头中携带有效的Authorization信息

#### 预期结果
- 返回状态码200
- 响应数据只包含"学习用品"分类的商品
- 每个商品的category字段均为"学习用品"

### 测试用例 1.3 - 关键词搜索商品

## 3. 兑换商品接口测试

### 测试用例 3.1 - 兑换商品成功

#### 前置条件
- 用户已登录，拥有有效的token
- 用户当前积分为500
- 系统中已存在ID为1001的商品，价格为100积分，库存充足

#### 测试步骤
1. 使用POST方法请求 `/api/shop/exchange` 接口
2. 请求头中携带有效的Authorization信息
3. 请求体包含兑换信息：
   ```json
   {
     "productId": 1001,
     "quantity": 2
   }
   ```

#### 预期结果
- 返回状态码200
- 响应数据包含兑换成功的订单信息，包括订单号、商品名称、数量、总积分
- 用户积分已减少200（100 x 2）
- 返回数据中包含兑换前后的积分余额

## 6. 管理接口测试 - 添加商品

### 测试用例 6.1 - 添加商品成功

#### 前置条件
- 用户已登录，拥有管理员权限
- 用户拥有有效的token

#### 测试步骤
1. 使用POST方法请求 `/api/shop/admin/products` 接口
2. 请求头中携带有效的Authorization信息
3. 请求体包含有效的商品数据：
   ```json
   {
     "name": "机械键盘",
     "description": "游戏专用机械键盘，青轴",
     "detailDescription": "采用德国原厂轴体，带背光，键位可编程，人体工学设计，使用寿命超过5000万次按键。",
     "price": 500,
     "image": "https://example.com/images/keyboard.jpg",
     "imageGallery": [
       "https://example.com/images/keyboard_1.jpg",
       "https://example.com/images/keyboard_2.jpg"
     ],
     "category": "电子产品",
     "stock": 20
   }
   ```

#### 预期结果
- 返回状态码200
- 响应数据包含添加成功的商品信息，包括新生成的商品ID
- 初始值：popularity=0，exchangeCount=0
- 创建时间createTime为当前时间

### 测试用例 6.2 - 添加商品参数缺失

#### 前置条件
- 用户已登录，拥有管理员权限
- 用户拥有有效的token

#### 测试步骤
1. 使用POST方法请求 `/api/shop/admin/products` 接口
2. 请求头中携带有效的Authorization信息
3. 请求体缺少必填参数name：
   ```json
   {
     "description": "游戏专用机械键盘，青轴",
     "price": 500,
     "image": "https://example.com/images/keyboard.jpg",
     "category": "电子产品",
     "stock": 20
   }
   ```

#### 预期结果
- 返回状态码400
- 返回参数错误信息，提示缺少必填参数name

### 测试用例 6.3 - 无权限添加商品

#### 前置条件
- 用户已登录，但没有管理员权限
- 用户拥有有效的token

#### 测试步骤

3. 请求体包含有效的商品数据
