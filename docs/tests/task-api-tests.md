# 任务API接口测试用例

## 1. 获取任务列表接口测试

### 测试用例 1.1 - 获取所有任务列表

#### 前置条件
- 用户已登录，拥有有效的token

#### 测试步骤
1. 使用GET方法请求 `/api/tasks` 接口
2. 请求头中携带有效的Authorization信息

#### 预期结果
- 返回状态码200
- 响应数据包含任务列表，包括进行中和已完成的任务
- 返回的JSON格式符合API文档规范
- 分页信息正确

## 2. 创建任务接口测试

### 测试用例 2.1 - 创建普通任务成功

#### 前置条件
- 用户已登录，拥有有效的token

#### 测试步骤
1. 使用POST方法请求 `/api/tasks` 接口
2. 请求头中携带有效的Authorization信息
3. 请求体包含有效的任务数据：
   ```json
   {
     "title": "完成数学作业",
     "total": 45,
     "tags": ["学习能力", "专注力"],
     "points": 15
   }
   ```
